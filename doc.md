# 前端变量命名助手 V1.0 需求文档（纯本地离线版）

**技术栈**：Python + PySide6
**核心约束**：全程本地运行、无任何网络请求、无云端接口调用、数据本地存储
**目标用户**：前端开发人员
**核心用途**：输入中文描述，快速生成**简短、语义准确、符合前端规范**的小驼峰变量名/函数名

---

## 一、整体产品定位

1. 轻量桌面小工具，常驻后台不占用过多系统资源，内存控制在 50MB 以内
2. 纯离线运行，所有翻译、分词、命名逻辑、词库均本地实现，断网完全可用
3. 聚焦编程场景，区别于通用翻译软件，输出名称精简、贴合前端编码习惯
4. 支持自定义扩充词库，长期使用可沉淀个人专属命名词汇

---

## 二、运行环境与打包要求

### 1. 运行环境

- 依赖：electrobun vue3
- 支持系统：Windows 10 / Windows 11（主流桌面系统）

### 2. 打包要求

- 打包为**单文件 `.exe`**，免安装，双击直接启动
- 打包后不附带冗余文件，词库、配置文件内嵌/同目录存放
- 启动速度快（冷启动≤1秒），后台静默运行无卡顿

---

## 三、核心功能模块（全本地实现）

### 模块1：基础输入与内容清洗

1. **输入区域**
   - 单行文本输入框，支持手动输入、`Ctrl+V` 粘贴中文描述
   - 输入限制：最大字符数 50（适配常规变量/函数语义描述）
   - 自动清洗：输入内容自动过滤**虚词**（的、了、和、与、及、之 等），不参与命名转换
2. **清空按钮**：一键清空输入框与结果区，快速开始下一次输入

### 模块2：命名模式切换（核心区分规则）

提供两种固定模式，本地逻辑判断生成对应格式名称，二选一单选：

1. **变量名模式（默认选中）**
   - 适用场景：`let/const` 定义普通变量、状态、标识、UI元素
   - 规则：名词/名词组合，纯小驼峰，单词数量控制在 1~4 个，拒绝冗长
   - 示例：用户列表 → `userList`、加载状态 → `loadingStatus`
2. **函数/方法名模式**
   - 适用场景：函数、事件处理、接口请求、操作行为
   - 规则：**动词开头 + 名词组合**，标准小驼峰，符合前端函数命名习惯
   - 示例：获取用户列表 → `getUserList`、关闭弹窗 → `closeModal`

### 模块3：本地词库匹配 & 分词转换（核心离线能力）

#### 3.1 底层依赖

- 中文分词：集成 `jieba` 本地分词库，对长文本自动拆分标准词汇
- 翻译映射：内置**分层本地词库（JSON格式）**，中文 → 英文编程词汇一一对应，无联网翻译

#### 3.2 词库设计（分层管理，全面可扩展）

词库分为5大类，覆盖前端95%以上编码场景，支持手动新增/编辑：

1. **通用基础名词**：用户、列表、数据、状态、弹窗、表单、按钮、菜单等
2. **通用动词（函数专用）**：获取、设置、打开、关闭、提交、删除、更新、处理等
3. **状态/布尔词汇**：显示、隐藏、加载、禁用、选中、完成等（适配 `isXXX` 格式）
4. **前端UI/组件词汇**：输入框、表格、侧边栏、图标、标签、分页等
5. **通用业务词汇**：订单、商品、支付、权限、角色、路由、缓存等

- 命名规则：统一使用行业通用**标准缩写**（配置=config、消息=msg、背景=bg），严控单词长度

#### 3.3 转换逻辑（全本地执行）

1. 对输入文本执行分词 → 过滤虚词 → 得到有效词汇列表
2. 逐词匹配本地词库，翻译成对应英文单词
3. 根据选中模式，拼接为**标准小驼峰格式**
4. 同一组词汇生成 3~5 个差异化推荐名称（组合微调、常用缩写替换）

#### 3.4 降级兜底规则

- 词汇完全匹配：优先使用词库标准译词
- 部分词汇匹配：已匹配词汇正常转换，未匹配词汇保留原中文（标注提示）
- 全词无匹配：结果区提示「暂无匹配词汇，请补充词库」

### 模块4：结果展示 & 一键复制

1. **结果展示区**
   - 列表形式展示 3~5 条推荐命名，每条独立条目
   - 区分样式：变量名/函数名用轻微色差区分，视觉清晰
2. **一键复制功能**
   - 点击任意一条命名，自动复制到系统剪贴板（本地剪贴板API，无网络）
   - 复制后弹出短时悬浮提示：「复制成功」（2秒后自动消失）
3. 支持鼠标悬停高亮，提升操作体验

### 模块5：本地词库管理（核心扩展能力，解决“词库全面性”）

纯本地编辑，修改实时生效，所有配置**本地文件持久化存储**：

1. **打开词库**：按钮触发，用系统默认文本编辑器打开词库 JSON 文件，支持手动批量增删改词汇
2. **快捷添加词汇**（简易弹窗）
   - 弹窗内两个输入框：中文词汇、对应英文编程词汇
   - 选择词汇分类（名词/动词/状态词）
   - 提交后自动写入词库、自动去重、即时生效，无需重启软件
3. **词库重置**：支持一键恢复为初始默认词库（防误修改）

### 模块6：历史记录（本地存储）

1. 自动记录**最近20条**输入内容 + 对应生成的命名结果
2. 历史记录本地文件存储，重启软件不丢失
3. 功能：点击历史条目可快速回填到输入框、一键清空全部历史

### 模块7：窗口辅助功能（适配编码场景）

1. **窗口置顶**：开关按钮，开启后工具窗口始终悬浮在最上层，编码时无需切换窗口
2. **最小化到托盘**：点击最小化按钮，程序缩至系统托盘，后台常驻；点击托盘图标恢复窗口
3. 窗口尺寸固定：小巧紧凑型窗口（宽400px / 高320px），不占用大量屏幕空间

## 四、界面视觉与样式规范（PySide6 美化，轻量化）

1. **整体风格**：现代极简风，默认**深色模式**（适配代码编辑器护眼习惯），可选浅色模式
2. **样式细节**
   - 所有控件添加圆角、轻微阴影，摒弃Qt原生老旧样式
   - 按钮/输入框 hover 状态有颜色变化反馈
   - 字体：系统等宽字体，字号适中，代码类文字清晰易读
3. **布局分区（从上至下）**
   1. 顶部：窗口标题 + 置顶开关 + 模式切换单选按钮
   2. 中上：文本输入框 + 清空按钮
   3. 中部：命名结果展示列表
   4. 中下：历史记录入口、复制提示区
   5. 底部：词库管理按钮、清空历史按钮

## 五、数据存储方案（全本地，文件存储）

所有数据以**普通文件**存放在程序同目录，无数据库、无云端同步：

1. `dict.json`：核心词汇库（中文-英文映射表，可手动编辑）
2. `history.json`：历史操作记录
3. `config.ini`：软件配置（窗口置顶状态、主题模式、窗口位置等）

- 特性：纯文本格式，可手动打开编辑，兼容所有Windows系统

## 六、命名生成强制规则（统一输出标准）

### 通用规则

1. 输出格式**强制为小驼峰（camelCase）**，禁止蛇形、大驼峰、全大写
2. 自动剔除冗余词汇、口语化词汇，名称尽量简短（单词数≤4）
3. 英文词汇使用前端行业通用写法与缩写，不使用生僻单词

### 分模式规则

1. **变量名模式**
   - 结构：`形容词/名词 + 名词`
   - 布尔状态类词汇：固定前缀 `is`（例：是否加载 → `isLoading`）
2. **函数名模式**
   - 结构：`动词 + 名词/名词组合`
   - 事件处理类：支持 `handle` 前缀（例：表单提交 → `handleSubmit`）

## 七、非功能需求

1. **离线约束**：程序运行全程**无任何网络请求**，禁用所有网络相关能力，防火墙无拦截行为
2. **性能要求**
   - 分词+词库匹配响应时间 ≤ 0.2秒
   - 后台托盘常驻时，CPU占用≈0，内存稳定≤50MB
3. **兼容性**：兼容 Windows 10 及以上 32/64位系统
4. **易用性**：所有操作单步完成，无复杂流程，新手开箱即用
5. **稳定性**：长时间后台常驻不崩溃、内存不泄漏、词库读写不损坏文件
